Filters

Controls:

  • 'r' - reset image
  • 'p' - switch images
  • 'z' - windmill
  • 'x' - cyberpunk
  • 'c' - edge detection*
  • 'v' - waves
  • 'b' - fisheye tunnel

*Please allow edge detection a few seconds to take effect.

Lessons learned:

  1. The pixels array of images in p5.js is not the same as that of Processing for java. In p5.js, the array is actually a one dimensional array with every 4 elements representing the RGBA values. The latter has one element corresponding to one color.
  2. A is from 0-255, not 0.0-1.0.
  3. loop(), noLoop(), redraw() exists. Both in p5.js and Processing for java. (Technically not really necessary but I found it because at some point I doubted my draw function was even running)
  4. The loadImage() function takes time to complete, we cannot call loadPixels() right after, but instead in a callback.
